# Adversarial Security Results

The adversarial pass identified missing MFA enforcement, duplicate-patient race exposure, permissive default table privileges, privacy-select paths, and weak abuse controls. Remediation added AAL2 enforcement, an advisory-lock duplicate fingerprint, restrictive default grants, explicit sensitive-table revocations, HMAC request fingerprints, idempotency, and a submission quota.

No claim is made that deployed policies resist live attacks until negative-role tests run against Supabase.
